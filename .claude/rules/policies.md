## Policies

- Do not introduce features, sections, or content that are absent from the reference image
- The objective is exact replication — no autonomous design "improvements" are permitted
- CSS classes or style tokens provided by the user must be incorporated without modification
- Keep code clear; avoid excessive abstraction. Inline Tailwind classes are acceptable
- Screenshot comparison feedback must be quantitative and specific:
  - ✓ `"Heading font-size is 32px; reference requires 38px"`
  - ✓ `"Card gap is 16px; specification requires 24px"`
  - ✗ `"The spacing looks off"`
